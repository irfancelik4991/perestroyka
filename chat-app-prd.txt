Gerçek Zamanlı Sohbet Uygulaması Product Requirements Document (PRD) Stack: Convex + TanStack Start Bu proje, yazilim gelistirmeyi ogretmek icin tasarlanmis egitim amacli bir uygulamadir. Convex (backend + realtime DB) ve TanStack Start (SSR + file-based routing) kullanilarak websocket, auth, oda yonetimi ve bildirim gibi gercek dunyada kullanilan konseptler ogretilir. 1. Proje Ozeti Bu uygulama, birden fazla kullanicinin gercek zamanli mesajlasabilecegi, odalara ayrilabilen ve online/offline durumun takip edilebildigi bir sohbet platformudur. Discord ya da Slack&apos;in cok kucuk bir versiyonu olarak dusunulebilir. Proje kasitli olarak orta zorlukta tutulmustur: baslangic seviyesinde sikici olmayacak kadar karmasik, ama yeni baslayanin kaybolmayacagi kadar sade. 2. Egitim Hedefleri Bu projeyi tamamlayan gelistirici su konulari ogrenmis olacak: Frontend (TanStack Start) File-based routing ile sayfa yapisi olusturma SSR (Server-Side Rendering) ve client-side rendering farki TanStack Router ile type-safe navigasyon Convex React hooks ile realtime veri baglama (useQuery, useMutation) Clerk entegrasyonu ile oturum yonetimi Backend (Convex) Convex sema tanimi ve reaktif query yazma Mutation ile veri yazma Action ile dis servis entegrasyonu (orn. bildirim gonderme) Convex&apos;in WebSocket altyapisinin nasil calistigini anlama Index tanimlayarak sorgu performansini artirma Genel Yazilim Muhendisligi Gercek zamanli sistem tasarimi (pub/sub modeli) Auth ve yetkilendirme akisi Optimistic updates ile UX iyilestirme Tip guvenligi (TypeScript end-to-end) 3. Teknoloji Stack Özellik Açıklama TanStack Start React tabanli SSR framework. File-based routing, server functions, type-safe navigation. TanStack Router TanStack Start ile birlikte gelir. URL state yonetimi ve nested layouts. Convex Backend as a Service. Reaktif veritabani, WebSocket, TypeScript-first sema, built-in auth. Clerk Kullanici kimlik dogrulamasi. Convex ile native entegrasyon, hazir UI bilesenleri. TypeScript Tum katmanlarda tip guvenligi. Convex sema&apos;dan frontend&apos;e kadar end-to-end. Tailwind CSS Utility-first CSS framework. Hizli prototipleme icin ideal. 4. Urun Ozellikleri 4.1 Kimlik Dogrulama Clerk ile email/sifre veya OAuth (Google) ile kayit ve giris Oturum bilgisi Convex&apos;e otomatik iletilir, her mutation/query&apos;de kimlik dogrulanir Giris yapilmamis kullanicilari login sayfasina yonlendiren route guard 4.2 Oda Yonetimi Kullanici yeni oda olusturabilir (isim + aciklama) Mevcut odalari listeleyen sidebar Odaya katilma ve ayrilma Oda sahibi odayi silebilir Her odanin aktif kullanici sayisi gosterilir 4.3 Mesajlasma Secili odada mesaj yazma ve gonderme Mesajlar gercek zamanli olarak guncellenir, sayfa yenilemek gerekmez Sonsuz scroll ile gecmis mesajlara erisim (pagination) Mesaj silme (yalnizca kendi mesajini) Optimistic update: gonderi butonu tiklandi mi, mesaj hemen gorunur 4.4 Online / Offline Durumu Kullanici uygulamadayken &apos;online&apos; olarak isareti Aktif kullanicilarin listesi sidebar&apos;da gosterilir Tarayici kapandiginda Convex presence mekanizmasi ile otomatik &apos;offline&apos; gecis 4.5 Bildirimler (Stretch Hedef) Kullanici baska bir odadayken mesaj gelirse tarayici bildirimi Okunmamis mesaj sayaci sidebar&apos;da rozet olarak gosterilir 5. Convex Veritabani Semasi Asagidaki tablolar Convex&apos;te tanimlanacak. convex/schema.ts dosyasinda defineSchema() ile olusturulur. Özellik Açıklama users Clerk&apos;ten gelen kullanici bilgileri. tokenIdentifier ile Clerk ID&apos;si saklanir. rooms Oda kayitlari. name, description, createdBy (user ID), createdAt alanlari. memberships Kullanici-oda iliskisi. userId + roomId + joinedAt. Hangi kullanici hangi odada. messages Mesaj kayitlari. roomId, userId, body, createdAt. roomId uzerine index zorunlu. presence Online durum. userId + roomId + lastSeen. 30 saniyede bir guncellenir. Ornek Sema Kodu // convex/schema.ts import { defineSchema, defineTable } from &apos;convex/server&apos;; import { v } from &apos;convex/values&apos;; export default defineSchema({ rooms: defineTable({ name: v.string(), description: v.optional(v.string()), createdBy: v.id(&apos;users&apos;), }), messages: defineTable({ roomId: v.id(&apos;rooms&apos;), userId: v.id(&apos;users&apos;), body: v.string(), }).index(&apos;by_room&apos;, [&apos;roomId&apos;]), }); 6. Sayfa Yapisi (Routes) TanStack Start file-based routing kullanir. Her dosya bir route olur. Özellik Açıklama / (root) Login sayfasina ya da /chat&apos;e yonlendirir. Auth durumuna gore. /login Clerk SignIn bileseni. Kullanici giris yapar, /chat&apos;e yonlendirilir. /chat Ana layout. Sol sidebar (oda listesi + online kullanicilar), sag icerik alani. /chat/$roomId Secili odanin mesaj ekrani. URL&apos;de oda ID&apos;si parametre olarak gelir. /chat/new Yeni oda olusturma formu. Klasor Yapisi src/ routes/ __root.tsx # HTML shell, Convex provider, Clerk provider index.tsx # / -&gt; /chat veya /login yonlendirme login.tsx # Clerk SignIn chat/ _layout.tsx # Sidebar + icerik alani layout index.tsx # Oda secilmemisse bos ekran new.tsx # Oda olusturma formu $roomId.tsx # Mesajlasma ekrani components/ Sidebar.tsx MessageList.tsx MessageInput.tsx RoomItem.tsx OnlineUsers.tsx convex/ schema.ts rooms.ts # listRooms, createRoom, deleteRoom messages.ts # listMessages, sendMessage, deleteMessage users.ts # getOrCreateUser, updatePresence 7. Onerilen Sprint Plani Bu plan bir kisi icin tasarlanmistir ve her sprint yaklasik bir haftalik calismayi kapsar. Sprint Süre Hedef Sprint 1 1 hafta Proje kurulumu. TanStack Start + Convex + Clerk entegrasyonu. Login/logout calisir hale getirilir. Sprint 2 1 hafta Oda yonetimi. Oda olusturma, listeleme, silme. Sidebar tasarimi. Sprint 3 1 hafta Mesajlasma. Mesaj gonderme, listeleme, silme. Gercek zamanli guncelleme. Sprint 4 1 hafta Online durum. Presence sistemi. Optimistic updates. UX polish. Sprint 5 Stretch Bildirimler. Okunmamis sayaci. Tarayici push notification. 8.
